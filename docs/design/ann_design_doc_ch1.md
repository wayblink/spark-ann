# Spark分布式ANN索引系统设计文档
## 第1章：概述与目标

### 1.1 项目背景

随着向量数据库和大规模机器学习应用的普及，对海量向量数据进行高效的相似性搜索成为关键需求。传统的精确最近邻搜索（KNN）在大规模数据集上性能不足，而现有的ANN（Approximate Nearest Neighbor）解决方案多为单机或专用系统，难以与现有的大数据生态深度集成。

本项目旨在基于Apache Spark构建一套分布式ANN索引系统，使企业能够在现有数据湖架构上直接进行高效的向量相似性搜索。

### 1.2 设计目标

#### 1.2.1 核心目标

**可扩展性**
- 支持PB级向量数据规模
- 支持高维向量（512-4096维）
- 水平扩展能力，随数据增长线性扩展

**精度优先**
- 保证高召回率（recall > 95%）
- 支持精度-性能权衡配置
- 提供fallback机制保证正确性

**非侵入性**
- 不改变现有数据文件分布
- 索引作为独立元数据层
- 兼容现有Spark数据处理流程

**灵活性**
- 支持多种索引类型（HNSW、IVF、LSH等）
- 支持多种距离度量（L2、Cosine、Inner Product）
- 可配置的查询参数

#### 1.2.2 长期目标

**与Apache Iceberg深度集成**
- 索引作为Iceberg表的一等公民
- 索引版本管理与数据版本同步
- 自动索引维护和优化

**智能查询优化**
- 基于代价的查询计划选择
- 自动索引选择
- 自适应参数调优

### 1.3 技术选型

**分布式计算框架**: Apache Spark 3.x
- 成熟的分布式计算能力
- 丰富的生态系统
- 与现有数据湖架构无缝集成

**索引算法**: HNSW (Hierarchical Navigable Small World)
- 业界最优的召回率和性能平衡
- 适合高维向量
- 支持增量更新

**存储格式**: Apache Parquet
- 列式存储，高效压缩
- 支持复杂嵌套结构
- 原生Spark支持

**元数据格式**: JSON
- 人类可读，便于调试
- 灵活的schema演进
- 与Iceberg元数据兼容

### 1.4 关键创新点

**两层索引架构**
- Global Layer：跨文件的粗粒度导航
- Local Layer：文件内的细粒度搜索
- 统一的HNSW图结构

**索引自描述**
- 完整的元数据定义
- 版本化管理
- 独立于计算引擎

**索引与数据分离**
- 保持数据文件不变
- 索引可独立构建和演进
- 支持多版本索引共存

**渐进式查询**
- 有索引时使用索引加速
- 无索引时自动降级到暴力搜索
- 对用户透明

### 1.5 非目标（本期不包含）

- 实时索引更新（后续版本支持）
- GPU加速（后续可选特性）
- 复杂过滤条件下的ANN（后续版本）：向量标量混合是个非常重要的使用方式，需要提前思考方案，要保证一期设计能够兼容这样的改变
- 多模态向量搜索（后续版本）：同上
